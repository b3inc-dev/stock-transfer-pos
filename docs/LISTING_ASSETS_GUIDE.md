# App Store リスティング用アセット・文案ガイド

**目的**: 公開アプリ（POS Stock）のリスティング入力時に参照するガイド。  
フィーチャー画像・スクリーンショットの内容、紹介文・詳細文の案、プライバシーポリシー・価格・デモ用ストアの考え方をまとめています。

---

## 1. フィーチャー画像（1600×900 px）

**何を映すとよいか**
- **メインの価値がひと目で分かる画面**を1枚で表現するのが理想です。
- 例：
  - **POSのホーム画面**で、4つのタイル（出庫/入庫/ロス/棚卸）が並んでいる様子
  - または **出庫・入庫のどちらか1つのモーダル**を開いた状態（商品リストや確定画面）
- 動画にする場合：インストール → POSアプリに追加 → タイルを開く → 簡単な操作（出庫1件 or 入庫1件）まで 2〜3分で流れを見せる形がおすすめです。
- **alt テキスト**は必須なので、「POSで出庫・入庫・ロス・棚卸ができる在庫アプリの画面」など短く記載します。

---

## 2. スクリーンショット（1600×900 px、3〜6枚）

**どんな画像を何枚用意するか（目安）**

| 枚目 | 内容の例 | 目的 |
|------|----------|------|
| 1 | **POS：アプリタイル一覧**（出庫・入庫・ロス・棚卸の4タイル） | 何ができるアプリかがすぐ分かる |
| 2 | **POS：出庫 or 入庫のコンディション／商品リスト** | 実際の操作画面を見せる |
| 3 | **管理画面：入出庫履歴 or ロス履歴 or 棚卸履歴** | Admin 側の確認・CSV など |
| 4 | **管理画面：設定 or 棚卸の商品グループ**（任意） | 設定や棚卸の柔軟性 |
| 5〜6 | 上記の別パターン（例：棚卸リスト、ロス登録画面） | 必要に応じて追加 |

**注意**
- ブラウザのアドレスバー・個人情報（店名・メール・電話）は写さない、またはぼかす。
- 「少なくとも1枚はアプリUI」「POS対応ならPOS画面も含める」という要件を満たすように、**管理画面＋POS** の両方を含めると安心です。

## 4. プライバシーポリシー（専用URL）

**どのドメインでどのページを作るか**

- **必須**なのは「**常時アクセスできる HTTPS の専用URL**」です。ドメインの種類は問いわれません。
- **よくあるパターン**：
  - **自社サイトがある場合**  
    `https://あなたのドメイン.com/privacy` や `https://あなたのドメイン.com/app-privacy` のような1ページを用意する。
  - **ドメインがない場合**  
    - 無料で使える **GitHub Pages**（例：`https://yourusername.github.io/pos-stock-privacy`）に1ページだけ作る。  
    - または **Render / Vercel など**で「Static Site」を1つ作り、`/privacy` や `/` にポリシーページを置く。

**ページに書く内容の例**

- 収集するデータ（例：Shopify から提供される店舗情報・在庫データ・スタッフ情報など）
- 利用目的（アプリの提供・在庫出庫・入庫・ロス・棚卸の処理）
- 保存期間・保存場所
- 第三者提供の有無（例：Shopify 以外に提供しない）
- お問い合わせ先（メールまたはフォーム）
- 改定日・改定内容（あれば）

テンプレートは「Shopify アプリ プライバシーポリシー テンプレート」などで検索すると参考例が見つかります。**法律的な最終確認は弁護士等に依頼してください。**

## 5. 価格設定（Free to install / プラン分け）

### 市場の価格帯の目安（POS・在庫系アプリ）

海外の類似アプリの月額目安はおおよそ次のとおりです。

| アプリ例 | 主な機能 | 月額目安（USD） |
|----------|----------|-----------------|
| Stocky（Shopify公式） | 在庫・発注・棚卸 | 無料〜 |
| Invo（複数ロケーション表示） | ロケーション別在庫表示 | 約 $5〜 |
| Synkro（複数ストア同期） | 在庫同期・商品数で段階 | $10〜$100 |
| QuickSync | ストア間在庫同期 | 約 $19〜 |
| Trunk（Stock Sync） | 在庫同期・バンドル | 約 $35〜 |
| Thrive（在庫管理） | 複数ロケーション・POS対応 | 約 $59〜 |

**このアプリ（出庫・入庫・ロス・棚卸の一括運用）の位置づけ**

- 出庫・入庫・ロス・棚卸を **POS＋管理画面** で一貫して扱える点が強みです。
- 競合は「同期系」が多く、「POSでの移管・入庫・ロス・棚卸の業務アプリ」としての直接比較は限られます。
- **月額 $15〜35（約 ¥2,200〜5,200）** が、有料で出す場合のひとつの目安になります。

### おすすめの金額帯（案）

- **単一プランで全機能**  
  - **月額 1,980円〜2,980円**（約 $13〜22）  
  - 導入のハードルを抑えつつ、価値に見合った水準。
- **プラン分けする場合**  
  - **ベーシック**（出庫・入庫のみ）：**月額 980円〜1,480円**（約 $7〜11）  
  - **プロ**（ロス・棚卸含む全機能）：**月額 2,480円〜3,980円**（約 $18〜30）  
  - 段階的に機能を足せるので、小規模店舗はベーシック、しっかり運用する店はプロ、という選び方がしやすくなります。

**決める際のポイント**

- **無料トライアル**（7日〜14日）を付けると、申し込みが増えやすいです。
- 価格は**価格セクションだけ**に書き、スクショ・紹介文には料金を入れない（Shopify のガイドラインに合わせる）。
- 日本円で表示するか、Shopify の請求を USD で行うかは、パートナー／請求の設定に従います。

### 基本的な選び方

- **無料で提供する場合**  
  パートナーダッシュボードの「価格」で **「Free to install」** を選び、価格セクション以外には料金を書かないようにします。
- **有料にする場合**  
  **Recurring charge（月額等）** または **One-time payment（買い切り）** のいずれかを選択し、Shopify の「アプリの請求」（Billing API または新しいアプリ請求）でプランと金額を定義します。

**「入庫・出庫のみ」と「全て」で分けられるか**

- **技術的には可能**です。  
  - 例：「ベーシック＝出庫・入庫のみ」「プロ＝出庫・入庫・ロス・棚卸すべて」のように、**プラン（料金）ごとに利用できる機能を変える**形にできます。
- その場合は次のような実装が必要です。
  1. パートナーで「料金プラン」（例：ベーシック ¥XXX/月、プロ ¥YYY/月）を用意する。  
  2. アプリ内で **Billing API** 等で「今のストアがどのプランか」を取得する。  
  3. ロス・棚卸の画面やタイルを、プロプランのときだけ表示（または有効）する。

## 6. デモ用ストア（POSアプリでも必要か）

**結論：あった方がよい（審査で求められることが多い）**

- 審査では「**アプリを実際に触れる環境**」が求められることがあります。POSアプリでも、**「どのストアにインストールされているか」「どこを開けばいいか」** を説明する必要があります。
- **デモ用ストア**には、次のような形で対応できます。
  - **開発ストアを1つ用意**し、そこに公開アプリ（POS Stock）をインストールする。
  - リスティングの「デモストアURL」に、その開発ストアのURL（例：`https://あなたの-dev-store.myshopify.com`）を記載する。
  - **説明文**に、例えば次のように書く：  
    「上記URLのストアに管理者としてログイン後、**POS → POSアプリ** から『出庫/在庫処理』『入庫/在庫処理』『ロス/在庫調整』『棚卸/在庫調整』のいずれかを開いてご確認ください。管理画面では **アプリ** メニューから入出庫履歴・ロス履歴・棚卸をご覧いただけます。」
- 「ECとしての販売」は必須ではなく、**アプリとPOSが使える状態になっている開発ストア**で十分です。審査用の**テスト用認証情報**（ログイン用メール・パスワードなど）も、同じストアで使うアカウントを案内するとスムーズです。
